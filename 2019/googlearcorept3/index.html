Google ARCore Unity-Part 3

zoonyoungc ♥ 7 October 2019 Unity 3 min

ARCore HelloAR Sample App Code (Android)


ARCore의 Customizing을 위한 구조 확인 (2019년 10월 기준)


Examples 구성

  • Part-1 에서 했던 방식으로 새 project를 만들어 보면 아래 경로에 샘플들이 모여있음 Assets/GoogleARCore/Examples/

  • 샘플 Scene의 종류는

    • HelloAR : 평평한 바닥 및 벽을 추적하여 기기에서 탭하여 오브젝트 생성
    • CloudAnchors : 한 개 이상의 기기가 동시에 같은 오브젝트를 보게 하는 방식
    • AugmentedFaces : 얼굴을 추적하여 Anchor 포인트 생성, 오브젝트 생성
    • AugmentedImages : 이미지를 추적하여 Anchor 포인트 생성, 오브젝트 생성
    • ObjectManipulation : HelloAR 에서 발전한 것으로 생성한 오브젝트를 상하좌우로 이동

  • 위의 방법들을 활용하여 증강현실(Augmented Reality)에 대한 사용자 경험을 만들어줌

  • 이 중에서 먼저 해본 HelloAR 과 AugmentedImages 의 커스터마이징을 위한 구조 확인


HelloAR

  • HelloAR 샘플 Scene을 확인해 보면 위와 같이 게임오브젝트들이 있음


게임 오브젝트
설 명
ARCore Device ARCore의 세션을 관리하는 Prefab으로서 내부에 First Person Camera 를 가지고서 사용자가
실제 세상을 디바이스로 촬영할 때 캡쳐되는 내용을 게임 월드 상에서 배경으로 사용하여
마치 실제 디바이스와 같이 움직이는 역할
Environmental Light 실제 디바이스가 캡쳐하는 내용들의 평균 픽셀 밝기 추정값을 사용하여
AR Scene의 내부 월드의 조명을 조정하는 것. (자세한 내용은 원문참조)
Point Cloud 특징점들에 대하여 points 를 만드는 Prefab
HelloAR Controller AR Scene 을 전체 관리하는 스크립트를 가진 게임 오브젝트.
여기서 참조하는 게임오브젝트는 First Person Camera 와 사용자가 동적으로 생성할 수 있는
Prefab 들로 구성되어있음(버전이 바뀌면서 Andy Android -> ARCorePawn 으로 Prefab들도 바뀜) 자세한 구조는 아래 항목에
Plane Discovery 사용자가 주변의 평평한 벽, 지면 등을 스캔하여 AR 경험을 줄 수 있도록 가이드 해주는 Prefab. 시간이 지나도 사용자가 이용을 못할 경우를 대비해 가이드와 도움말이 버튼 형식으로 생성됨


Code

     HelloAR Scene 내부에 있는 Hello Controller 의 HelloARController 스크립트
     중요 사항은 Update() 함수에 있음

  • 모션 트래킹(동작 추적)은 평면 감지 기능을 제공함.

    • Motion Tracking 은 ARCore 디바이스가 현실 세계에서 움직일때 카메라로
      특징점들을 만들어 내면서 주변을 맵핑하여 디바이스의 위치와 방향을 추정함
    • ARCore 내부의 가상 카메라와 실제 디바이스의 카메라를 정렬함으로써 가상으로
      생성하는 3D 컨텐츠를 현실세계에 생성하는 것처럼 보이게 만듬
    • Update() 함수 내부에 _UpdateApplicationLifecycle() 이 Update 주기 동안 계속 모션 추적 확인
    • HelloARController.cs 의 _UpdateApplicationLifecycle() 함수의 일부

  • 실제 디바이스의 카메라로 ARCore 는 평면 감지를 시도하고 찾지 못하면 계속 Plane Discovery 게임 오브젝트가 실행되어 가이드 UI를 보여줌.

  • 가상 오브젝트, 3D 컨텐츠 등을 배치 하기 위한 사용자 입력 평면 감지 이후에 Update() 함수는 사용자 입력을 처리.

    • 디바이스의 화면에 탭을 하게 되면, Frame.Raycast() 는 내부에 선을 쏴서 사용자가 특징점을 탭 했는지 감지함.
    • TrackableHit 함수는 사용자가 탭 한 위치에 Prefab으로 등록한 가상의 오브젝트를 인스턴스화 함(생성).
    • 탭 한 위치에 Anchor 가 생성되어 가상 오브젝트를 고정하는 역할을 하며
    • Anchor 와 가상 오브젝트가 더 이상 필요하지 않다면 내부에 있는 Destroy 함수를 호출하여 ARCore 추적 기능을 중지 시킬수 있음.

    • 인스턴스화되는 Prefab을 수정하여 내가 원하는 오브젝트를 배치시킬수 있음.
      ※ 원문 상에는 가상의 오브젝트가 Andy 인데 현재 버전에서는 ARCorePawn 임.

    • HelloARController.cs 의 Update() 부분, 원문과는 내용이 다르기 때문에 실제 코드에서 가져옴.
